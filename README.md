#Para realizar commits utilizar estes prefixos no comentário:

feat: Uma nova funcionalidade ou um novo recurso.

Exemplo: feat: adiciona autenticação de usuário

fix: Uma correção de bug.

Exemplo: fix: corrige o bug de login na página inicial

docs: Mudanças apenas na documentação.

Exemplo: docs: atualiza o README com instruções de instalação

style: Mudanças que não afetam a lógica do código (formatação, espaços em branco, ponto e vírgula).

Exemplo: style: formata o código com o Prettier

refactor: Uma mudança de código que não adiciona uma nova funcionalidade nem corrige um bug. Geralmente, é para melhorar a estrutura.

Exemplo: refactor: extrai a lógica de validação para uma função separada

test: Adiciona ou corrige testes.

Exemplo: test: adiciona testes de unidade para a função de login

chore: Mudanças de rotina, como a atualização de dependências ou outras tarefas de build que não afetam o código de produção.

Exemplo: chore: atualiza a versão do docker-compose
